## 1. 项目概览

### 1.1 项目定位

Hermes Agent 是一个**自我改进的 AI 代理**，其核心特性：

| 特性 | 说明 |
|------|------|
| **闭环学习** | 自动从经验创建技能，使用中改进，跨会话记忆，兼容 [agentskills.io](https://agentskills.io) 标准 |
| **多入口接入** | CLI、TUI、桌面应用（Electron）、消息网关（Telegram、Discord、Slack 等 20+ 平台） |
| **任意模型支持** | Nous Portal、OpenRouter、OpenAI、Anthropic、Google、本地 Ollama 等 |
| **任意环境运行** | 6 种终端后端：本地、Docker、SSH、Singularity、Modal、Daytona |
| **调度自动化** | 内建 cron 调度器，可向任意平台交付任务 |
| **并行委托** | 生成隔离子代理处理并行工作流 |

### 1.2 技术栈

- **后端**：Python 3.11+（最高 3.13）
- **TUI**：TypeScript + Ink（React 终端框架）
- **桌面应用**：Electron + React + `@assistant-ui/react`
- **Web 控制台**：React + Vite
- **文档站点**：Docusaurus
- **打包**：setuptools + uv（Astral 的 Python 包管理器）
- **Nix 支持**：提供 flake.nix 用于 NixOS

### 1.3 许可证

MIT 许可证 — 见 [LICENSE](LICENSE)。

### 2.3 Agent 主循环

核心循环位于 `run_agent.py` 的 `run_conversation()` 方法中：

```python
while (api_call_count < self.max_iterations and self.iteration_budget.remaining > 0) \
        or self._budget_grace_call:
    if self._interrupt_requested: break
    response = client.chat.completions.create(
        model=model, messages=messages, tools=tool_schemas
    )
    if response.tool_calls:
        for tool_call in response.tool_calls:
            result = handle_function_call(tool_call.name, tool_call.args, task_id)
            messages.append(tool_result_message(result))
        api_call_count += 1
    else:
        return response.content
```

消息遵循 OpenAI 格式：`{"role": "system/user/assistant/tool", ...}`。推理内容存储在 `assistant_msg["reasoning"]`。

## 3. 核心模块详解

### 3.1 AIAgent 类（[run_agent.py](file:///d:/project/test/hermes-agent/run_agent.py)）

```python
class AIAgent:
    def __init__(self,
        base_url: str = None,
        api_key: str = None,
        provider: str = None,
        api_mode: str = None,              # "chat_completions" | "codex_responses"
        model: str = "",                   # 空 → 后续从配置解析
        max_iterations: int = 90,          # 工具调用迭代上限
        enabled_toolsets: list = None,
        disabled_toolsets: list = None,
        quiet_mode: bool = False,
        save_trajectories: bool = False,
        platform: str = None,              # "cli", "telegram" 等
        session_id: str = None,
        skip_context_files: bool = False,
        skip_memory: bool = False,
        credential_pool=None,
        # ... 其他 ~60 个参数
    ): ...

    def chat(self, message: str) -> str:
        """简单接口——返回最终响应字符串。"""

    def run_conversation(self, user_message, system_message=None,
                         conversation_history=None, task_id=None) -> dict:
        """完整接口——返回 dict 含 final_response + messages。"""
```

### 3.2 工具注册系统（[tools/registry.py](file:///d:/project/test/hermes-agent/tools/registry.py)）

```python
from tools.registry import registry

def example_tool(param: str, task_id: str = None) -> str:
    return json.dumps({"success": True, "data": "..."})

registry.register(
    name="example_tool",
    toolset="example",
    schema={"name": "example_tool", "description": "...", "parameters": {...}},
    handler=lambda args, **kw: example_tool(param=args.get("param", "")),
    check_fn=lambda: bool(os.getenv("EXAMPLE_API_KEY")),
    requires_env=["EXAMPLE_API_KEY"],
)
```

**关键约束**：所有 handler 必须返回 JSON 字符串。

### 3.3 斜杠命令注册表（[hermes_cli/commands.py](file:///d:/project/test/hermes-agent/hermes_cli/commands.py)）

所有斜杠命令集中定义在 `COMMAND_REGISTRY` 列表中，下游消费者自动派生：

## 4. 安装指南

### 4.1 Linux / macOS / WSL2 / Termux

```bash
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
```

### 4.2 Windows 原生（PowerShell）

```powershell
iex (irm https://hermes-agent.nousresearch.com/install.ps1)
```

安装器会自动处理：uv、Python 3.11、Node.js、ripgrep、ffmpeg，以及一个**便携的 Git Bash**（MinGit，无需管理员权限）。

### 4.3 安装后

```bash
source ~/.bashrc        # 重新加载 shell（或 source ~/.zshrc）
hermes                  # 开始聊天
```

### 4.4 手动安装（开发者）

```bash
# 使用 Astral 的 uv
curl -LsSf https://astral.sh/uv/install.sh | sh
uv venv ~/.hermes/venvs/hermes-dev --python 3.11
source ~/.hermes/venvs/hermes-dev/bin/activate
uv pip install -e ".[all,dev]"
scripts/run_tests.sh
```

### 4.5 从 OpenClaw 迁移

```bash
hermes claw migrate              # 交互式迁移（完整预设）
hermes claw migrate --dry-run    # 预览
hermes claw migrate --preset user-data  # 不含密钥
hermes claw migrate --overwrite  # 覆盖冲突
```

迁移内容：SOUL.md、记忆、技能、命令白名单、消息设置、API 密钥、TTS 资产、AGENTS.md。

### 8.4 Footprint Ladder（新能力决策阶梯）

选择**最高（最小足迹）**能正确解决问题的层级：

1. **扩展现有代码**——零新表面
2. **CLI 命令 + 技能**——零模型工具足迹（订阅、调度、服务设置的默认选择）
3. **服务门控工具（`check_fn`）**——前置条件未配置时零足迹
4. **插件**——在 `~/.hermes/plugins/` 或 pip 包中
5. **MCP 服务器（在目录中）**——零永久核心 schema 足迹
6. **新核心工具**——最后手段，仅当能力是基础且普遍有用时

### 8.5 添加核心工具

```python
# tools/your_tool.py
import json, os
from tools.registry import registry

def check_requirements() -> bool:
    return bool(os.getenv("EXAMPLE_API_KEY"))

def example_tool(param: str, task_id: str = None) -> str:
    return json.dumps({"success": True, "data": "..."})

registry.register(
    name="example_tool",
    toolset="example",
    schema={"name": "example_tool", "description": "...", "parameters": {...}},
    handler=lambda args, **kw: example_tool(param=args.get("param", "")),
    check_fn=check_requirements,
    requires_env=["EXAMPLE_API_KEY"],
)
```

然后**必须**在 [toolsets.py](file:///d:/project/test/hermes-agent/toolsets.py) 的 `_HERMES_CORE_TOOLS` 或新工具集中注册。

### 9.4 Curator（技能生命周期）

后台技能维护系统，追踪 agent 创建的技能使用情况并自动归档陈旧技能。

**用户不会丢失技能**——归档到 `~/.hermes/skills/.archive/`，可恢复。

CLI 命令：
```bash
hermes curator status     # 查看状态

hermes curator prune
hermes curator backup

```

**不变量**：
- Curator 只处理 `created_by: "agent"` 来源的技能——bundled + hub 安装的技能不受影响
- 从不删除；最大破坏性操作是归档
- 固定的技能豁免所有自动转换和 LLM 审查

配置（`curator:` 章节）：`enabled`、`interval_hours`、`min_idle_hours`、`stale_after_days`、`archive_after_days`、`backup.*`。

## 12. 定时任务（Cron）

### 12.1 命令

```bash
hermes cron list       # 列出任务
hermes cron add        # 添加任务
hermes cron edit       # 编辑
hermes cron pause/resume
hermes cron run        # 立即运行
hermes cron remove
hermes cron status     # 检查调度器运行状态
```

会话内：`/cron` 斜杠命令。

### 12.2 调度格式

- **持续时间**：`"30m"`、`"2h"`、`"1d"`
- **"every" 短语**：`"every 2h"`、`"every monday 9am"`
- **5 字段 cron 表达式**：`"0 9 * * *"`
- **ISO 时间戳（一次性）**：`"2026-06-01T09:00:00Z"`

### 12.3 任务字段

| 字段 | 说明 |
|------|------|
| `skills` | 加载特定技能 |
| `model` / `provider` | 模型覆盖 |
| `script` | 预运行数据收集脚本，stdout 注入到提示；`no_agent=True` 时脚本即整个任务 |
| `context_from` | 链式任务：将任务 A 的最后输出链接到任务 B 的提示 |
| `workdir` | 在特定目录运行（加载其 `AGENTS.md`/`CLAUDE.md`） |
| 多平台交付 | 投递到多个平台 |

### 12.4 硬化不变量

- **3 分钟硬中断**——防止失控代理循环占用调度器
- Catchup 窗口：任务周期的一半，限制在 120s–2h
- Grace 窗口：一次性任务错过的触发时间有 120s 宽限
- 文件锁 `~/.hermes/cron/.tick.lock` 防止跨进程重复 tick
- Cron 会话默认 `skip_memory=True`

## 13. 委托与并行子代理

### 13.1 两种形式

- **单任务**：`goal`（+ 可选 `context`、`toolsets`）
- **批量（并行）**：`tasks: [...]`，每个任务获得独立子代理并发运行
  - 并发上限：`delegation.max_concurrent_children`（默认 3）

### 13.2 角色

| 角色 | 能力 |
|------|------|
| `role="leaf"`（默认） | 聚焦工作者。**不能**调用 `delegate_task`、`clarify`、`memory`、`send_message`、`execute_code` |
| `role="orchestrator"` | 保留 `delegate_task`，可生成自己的工作者。由 `delegation.orchestrator_enabled`（默认 true）门控，受 `delegation.max_spawn_depth`（默认 2）限制 |

### 13.3 配置

```yaml
delegation:
  max_concurrent_children: 3
  max_spawn_depth: 2
  child_timeout_seconds: ...
  orchestrator_enabled: true
  subagent_auto_approve: ...
  inherit_mcp_toolsets: ...
  max_iterations: ...
```

### 13.4 持久性规则

后台 `delegate_task` 与当前回合分离但仍为进程本地。需要跨进程重启的工作应使用 `cronjob` 或 `terminal(background=True, notify_on_complete=True)`。

## 17. 插件系统

### 17.1 通用插件（[hermes_cli/plugins.py](file:///d:/project/test/hermes-agent/hermes_cli/plugins.py)）

`PluginManager` 从 `~/.hermes/plugins/`、`./.hermes/plugins/` 和 pip entry points 发现插件。

每个插件暴露 `register(ctx)` 函数，可以：
- 注册 Python 回调生命周期钩子：`pre_tool_call`、`post_tool_call`、`pre_llm_call`、`post_llm_call`、`on_session_start`、`on_session_end`
- 通过 `ctx.register_tool(...)` 注册新工具
- 通过 `ctx.register_cli_command(...)` 注册 CLI 子命令

### 17.2 内置插件目录

- `plugins/memory/` — 记忆提供商插件
- `plugins/context_engine/` — 上下文引擎插件
- `plugins/model-providers/` — 推理后端插件
- `plugins/kanban/` — 多代理看板
- `plugins/image_gen/` — 图像生成提供商
- `plugins/web/` — 网络搜索提供商（exa、xai）
- 其他：google_meet、spotify、disk-cleanup 等

### 17.3 模型提供商插件

每个推理后端（openrouter、anthropic、gmi、deepseek、nvidia 等）作为插件发布。**懒加载**——首次调用 `get_provider_profile()` 或 `list_providers()` 时扫描。

扫描顺序：
1. Bundled：`<repo>/plugins/model-providers/<name>/`
2. User：`$HERMES_HOME/plugins/model-providers/<name>/`
3. Legacy：`<repo>/providers/<name>.py`

用户插件同名覆盖 bundled——`register_provider()` 是最后写入胜出。

### 17.4 重要政策

- **插件不得修改核心文件**（`run_agent.py`、`cli.py`、`gateway/run.py`、`hermes_cli/main.py` 等）
- **不再接受新的内存提供商**作为内置——必须作为独立插件仓库发布
- **不再接受第三方产品插件**——observability、SaaS 连接器、分析等必须作为独立仓库发布

### 17.5 编写插件

```python
# ~/.hermes/plugins/myplugin/__init__.py
from hermes_cli.plugins import PluginContext

def register(ctx: PluginContext):
    # 注册工具
    ctx.register_tool(name="my_tool", schema={...}, handler=...)

    # 注册 CLI 子命令
    ctx.register_cli_command(name="mycommand", ...)

    # 生命周期钩子
    @ctx.hook("pre_tool_call")
    def my_hook(tool_name, args):
        ...
```

## 19. Profile 多实例管理

### 19.1 概念

Hermes 支持 **profiles**——多个完全隔离的实例，每个有自己的 `HERMES_HOME` 目录（配置、API 密钥、记忆、会话、技能、网关等）。

核心机制：`_apply_profile_override()` 在 [hermes_cli/main.py](file:///d:/project/test/hermes-agent/hermes_cli/main.py) 中在任何模块导入前设置 `HERMES_HOME`。

### 19.2 使用 Profile

```bash
hermes -p coder           # 使用 "coder" profile
hermes -p coder profile list    # 列出所有 profile（无论哪个活跃）
```

### 19.3 Profile 安全代码规则

1. **所有 HERMES_HOME 路径使用 `get_hermes_home()`**（从 `hermes_constants` 导入）：
   ```python
   # 正确
   from hermes_constants import get_hermes_home
   config_path = get_hermes_home() / "config.yaml"

   # 错误——破坏 profile
   config_path = Path.home() / ".hermes" / "config.yaml"
   ```

2. **用户可见消息使用 `display_hermes_home()`**：
   ```python
   from hermes_constants import display_hermes_home
   print(f"Config saved to {display_hermes_home()}/config.yaml")
   ```

3. **网关平台适配器应使用 token 锁**——防止两个 profile 使用相同凭证

4. **Profile 操作是 HOME 锚定的**——`_get_profiles_root()` 返回 `Path.home() / ".hermes" / "profiles"`

## 20. 开发与贡献

### 20.1 开发环境

```bash
# 使用标准安装器，然后从完整 git checkout 工作
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
cd "${HERMES_HOME:-$HOME/.hermes}/hermes-agent"
uv pip install -e ".[all,dev]"
scripts/run_tests.sh
```

### 20.2 测试

**始终使用 `scripts/run_tests.sh`**——不要直接调用 `pytest`：

```bash
scripts/run_tests.sh                                  # 完整套件，CI 一致
scripts/run_tests.sh tests/gateway/                   # 一个目录
scripts/run_tests.sh tests/agent/test_foo.py::test_x  # 一个测试
scripts/run_tests.sh -v --tb=long                     # 传递 pytest 标志
```

每个测试文件在新生成的 Python 子进程中运行（subprocess-per-test-file 隔离）。

### 20.3 测试规则

- **不要写变化检测器测试**——不要断言会随数据更新而失败的快照
- 写**行为契约**测试，而非快照
- 测试不得写入 `~/.hermes/`（`_isolate_hermes_home` autouse fixture 重定向到临时目录）

### 20.4 贡献标准

**想要的**：
- 修复真实 bug（在当前 `main` 上复现症状）
- 在边缘扩展（新平台、渠道、提供商、模型）
- 将 god-file 重构为干净模块
- 保持核心狭窄
- 行为契约而非快照
- E2E 验证而非仅绿色单元 mock

**不想要的**：
- 投机性基础设施（无具体消费者的钩子）
- 用于非密钥配置的新 `HERMES_*` env 变量
- 当 terminal + file 已能胜任时的核心工具
- 破坏其保护功能的"修复"
- 未经选择性加入的出站遥测
- 第三方产品集成到核心树

### 20.5 依赖固定政策

所有依赖必须有上界以限制供应链攻击面：

| 来源类型 | 处理方式 | 示例 |
|---------|---------|------|
| PyPI 包 | `>=floor,<next_major` | `"httpx>=0.28.1,<1"` |
| Git URL | commit SHA | `git+https://...@<40-char-sha>` |
| GitHub Actions | commit SHA + 注释 | `uses: actions/checkout@<sha>  # v4` |
| CI-only pip | `==exact` | `pyyaml==6.0.2` |

### 20.6 已知陷阱

- **不要硬编码 `~/.hermes` 路径**——使用 `get_hermes_home()` / `display_hermes_home()`
- **不要引入新的 `simple_term_menu` 用法**——使用 `hermes_cli/curses_ui.py`
- **不要在 spinner/display 代码中使用 `\033[K`**——在 `prompt_toolkit` 的 `patch_stdout` 下泄漏为字面文本
- **`_last_resolved_tool_names` 是进程全局**——`delegate_tool.py` 在子代理执行周围保存/恢复
- **不要在 schema 描述中硬编码跨工具引用**

## 21. 常见问题排查

### 21.1 `hermes doctor`

```bash
hermes doctor    # 检查配置和依赖
```

### 21.2 查看日志

```bash
hermes logs --follow                # 跟踪日志
hermes logs --level WARNING         # 级别过滤
hermes logs --session <session_id>  # 会话过滤
```

日志位置：`~/.hermes/logs/`（`agent.log` INFO+、`errors.log` WARNING+、`gateway.log`）。

### 21.3 Windows Defender / 杀毒软件标记 `uv.exe`

误报。`uv` 是 Astral 的 Rust Python 包管理器。

验证：
```powershell
$uv = "$env:LOCALAPPDATA\hermes\bin\uv.exe"
$ver = (& $uv --version).Split(' ')[1]
$zip = "$env:TEMP\uv.zip"
Invoke-WebRequest "https://github.com/astral-sh/uv/releases/download/$ver/uv-x86_64-pc-windows-msvc.zip" -OutFile $zip -UseBasicParsing
gh attestation verify $zip --repo astral-sh/uv
```

白名单：
```powershell
# PowerShell 管理员
Add-MpPreference -ExclusionPath "$env:LOCALAPPDATA\hermes\bin"
```

### 21.4 config.yaml 解析失败

如果 YAML 无法解析，`load_config()` 会回退到 `DEFAULT_CONFIG`，所有用户覆盖被忽略。错误会：
- 在 stderr 显示一次警告
- 在 `agent.log` / `errors.log` 记录 WARNING
- 损坏文件备份到 `config.yaml.corrupt.<timestamp>.bak`

### 21.5 Prompt 缓存破坏

如果对话成本异常高，检查是否：
- 在会话中改变了过去上下文
- 在会话中切换了工具集
- 在会话中重新加载了记忆或重建了系统提示

这些都是禁止的——唯一的例外是上下文压缩。

### 21.6 网关消息无法到达运行中的代理

网关有**两个消息守卫**：
1. base adapter 在 `session_key in self._active_sessions` 时将消息排队到 `_pending_messages`
2. gateway runner 在到达 `running_agent.interrupt()` 前拦截 `/stop`、`/new`、`/queue`、`/status`、`/approve`、`/deny`

任何必须在代理阻塞时到达 runner 的新命令必须**绕过两个守卫**并内联分发。

### 21.7 平台特定

- **Termux/Android**：使用 `.[termux]` extra（`.[all]` 包含 Android 不兼容的语音依赖）
- **Windows**：原生完全支持；`%LOCALAPPDATA%\hermes` 下；WSL2 在 `~/.hermes`
- **macOS/Linux**：标准安装路径 `~/.hermes`
